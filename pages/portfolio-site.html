Portfolio Project

Overview

This portfolio website was designed to showcase my projects, skills, and experience in a way that feels clean, organized, and easy to navigate. The goal was to create a space where someone can quickly understand what I’ve built, how I approach development, and the type of work I enjoy doing.


Rather than just listing projects, I wanted each one to have its own dedicated page that gives more context into the design decisions, tools used, and development process behind it. This makes the site feel more like a collection of real work instead of just a gallery.


A big focus of this project was creating a layout that stays consistent across different screen sizes while still feeling visually balanced. I paid attention to spacing, alignment, and structure so everything feels intentional and easy to follow. Overall, the site reflects both my technical skills and the way I approach designing user friendly experiences.


You can view the source code on GitHub.

Tools

  • HTML5
  • CSS3
  • Flexbox
  • CSS Grid
  • Responsive Design
  • Git & GitHub

Process

This project started with planning out the structure of the site, deciding what pages were needed and how users would move between them. I wanted the layout to feel simple and consistent, with sections like Home, Projects, and Contact clearly defined so navigation feels natural.


From there, I built the site page by page, starting with the main layout and then adding individual components like project cards, detail pages, and the contact section. A big part of the process was testing spacing, alignment, and layout choices to make sure the design looked clean and held up across different screen sizes.


As the site came together, I kept refining smaller details like button styles, section spacing, image placement, and overall consistency. I also organized the styling into separate CSS files so the project would stay easier to manage as it grew.


Overall, this project was both a design and development exercise. I focused not just on making the site functional, but on making it feel polished, intentional, and easy for someone to explore.
